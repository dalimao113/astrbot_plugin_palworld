# CLAUDE.md · 项目长期开发方向

> 本文件是 `astrbot_plugin_palworld` 的长期规则与约束,随 Git 持久保存。
> 每次会话(含 /clear、/compact、重开)都应先读本文件,再动手。
> 专项规则见 `.claude/rules/`,当前进度见 `docs/DEVELOPMENT_ROADMAP.md`。

## 项目定位

- AstrBot 幻兽帕鲁(Palworld)服务器查询与管理插件。
- 主要用于**几个朋友的私人服务器联机**,不面向大型公共群/多租户/商业场景。
- 优先级:**游戏数据准确 > 1.0 存档兼容 > 功能稳定 > 操作体验 > 游戏 UI 还原**。
- 不按大型公共机器人/企业级系统过度设计;不为理论高并发或减少代码行数做无收益重构。
- 不随意改变已有指令、别名、配置名、用户行为。

## 三套独立主题(强制)

插件长期保留三套**独立、可切换**主题,通过配置 `card_style` 选择:

- `fantasy` — 现有二次元奇幻玻璃主题。
- `pixel` — 现有像素羊皮纸主题。
- `ingame` — 《幻兽帕鲁》游戏原生主题(**已落地:56 卡全覆盖,真实游戏图标 + pak 真配色,v1.9.0**;视觉精校待截图)。

约束:
- 三套主题**各自独立**的视觉语言:**背景 / 面板 / 边框 / 字体 / 配色 / 排版 / 卡片组件样式**互不覆盖、互不改动。
- **真实游戏语义图标是「共享游戏素材层」,三主题共用,不再是 ingame 专属**(见下「主题与图标分层」)。
- 三套主题**共用**服务器查询/图鉴/存档/配种等业务逻辑;**图标文件只保留一份**,经 `render/assets.py` 的 manifest 解析。
- **不得为三套主题复制三份 Handler、业务代码或图标文件**。
- 三套主题必须拥有**相同的模板键集合**(`TEMPLATE_KEYS`)。
- 改某一主题的**布局/面板/边框**时不得改动另两套;但接入/更新**游戏语义图标**是三主题共享的,一次接入自动供三套使用。
- 素材缺失时**安全回退**(fantasy/pixel 回退 Emoji、ingame 回退中性占位),不得让指令异常;未知 `card_style` 值回退到 fantasy。
- 新增功能时必须考虑三套主题是否都有对应展示。

## 主题与图标分层(强制)

- **主题只决定图标的展示方式**(fantasy 奇幻光晕/圆托、pixel 像素边框/`image-rendering`、ingame 游戏槽位),**不决定图标来源**。
- **游戏语义图标(三主题共享)**:属性/工作适性/货币/HP/攻击/防御/SAN 等数值/稀有度/性别/闪光/头目/塔主/突变/浓缩/觉醒/物品/材料/科技/建筑/武器/技能/技能果实/蛋糕/帕鲁蛋/植入体/词条等级——**只要游戏客户端有正式图标,fantasy/pixel/ingame 都必须优先用真实游戏图标**,不得再用 🔥💧⚡⛏️🔨💰 等 Emoji 代替。
- **UI 组件纹理(仅 ingame)**:游戏窗口/切角面板/物品槽边框等 `component_uris()` 纹理**只供 ingame**,不得套到 fantasy/pixel(那会破坏其独立视觉)。语义图标 ≠ UI 组件纹理,两者分开。
- **取图接口**:`assets.game_icon(key)` / `img(key, style)` 三主题共享游戏图标;`game_icon_map(style)` 供模板 `{{ icons.* }}`(renderer 对三主题都注入)。manifest 资产字段兼容 `game`(新)/`ingame`(旧)。
- 覆盖表见 `docs/THEME_ICON_COVERAGE.md`(语义键/游戏有无原图/资产路径/三主题是否使用/缺失原因/回退)。

## ingame 主题目标

- 目标 **85%–95%** 游戏原生 UI 视觉还原。
- 不机械照搬 1920×1080 横屏;适配 QQ 群**手机竖屏**阅读(允许重排,保留游戏视觉语言)。
- 尽量用**游戏文件里的真实**图标/UI 纹理/面板/边框/状态图标/稀有度样式。
  - 游戏文件负责:真实数据与素材。
  - 游戏截图负责:布局/间距/字号/颜色/信息层级/选中状态参考。
- **不得用 AI 生成图标冒充游戏原始图标**;未取得真实素材必须明确标记缺失。
- **未获得参考截图,不得凭空设计后声称高度还原**。

## 图标 Emoji 规则(三主题统一)

- 不能只替换部分页面的 Emoji;必须**审计所有用户可见卡片**(模板静态 + Python 动态注入两类都查)。
- **三主题**中只要游戏有对应正式图标就**必须**用游戏图标:属性/状态/货币/数值/技能/科技/材料/稀有度/性别/闪光/头目/塔主/突变/浓缩/觉醒/工作适性等。
- **fantasy/pixel 不再为「游戏已有图标的概念」保留 Emoji**;仅在游戏无对应图标、或素材文件意外缺失时才回退 Emoji。
- 游戏中**不存在**的插件扩展概念(CPU/内存/Docker/QQ/REST/网络延迟/服务器在线离线/备份/审计/自检/管理员/搜索/成功失败/通知等)不得套用语义错误的游戏图标:fantasy 用 Emoji/奇幻图标、pixel 用像素图标、ingame 用中性线性 SVG,明确标为"插件扩展图标"。
- 用户自己输入的 Emoji(昵称/公告/聊天)**不得删除**。
- 新接入游戏语义图标时:进 manifest 一次,即自动供三主题使用(不改三份)。

## 游戏数据规则

- **不得把 DataTable 总行数直接当官方图鉴数**;区分可收集/亚种/Boss/剧情/内部实体。
  - 现口径:**287 官方可收集 / 289 数据实体**(差 2 = 枯星龙剧情 boss + 花叶泥泥变种),已用 `is_collectible` 等字段标记。
- 生成数据记录 `game_version`/`steam_build_id`/`generated_at`/`source`。
- **官方未公开的数值/概率不得猜测**,标注"游戏未公开"或"仅说明机制"。
- **概率/权重必须带来源**:显示任何概率/权重前,须有 build ID + 资产路径(DataTable/Curve/Blueprint 字段)+ 原始值 + 换算;拿不到就显示"当前构建未提取到官方准确数值",不引用论坛/猜测。
- **业务实体以稳定 ID 为主键**(item_id/pal_dev_name/mission_id);中文名索引不得静默覆盖同名实体,重名时返回候选列表或优先可收集本体。名称一律去首尾空格。
- **地图数据必须带 `map_id` + 原始世界坐标**;百分比坐标由世界坐标 + 该地图 transform 派生,不作唯一真值;归类前不得 clamp 坐标。一个帕鲁可同时属于多个地图。
- **详情页不得硬截断**文本(`[:N]`);列表页可给摘要字段,详情须完整 + 自动换行。用户可见文本统一走 `utils.text.clean_text`(解析 `<itemName>`、去零宽/控制字符、占位"zh-hans text"/"None"视为空),不在各 handler 各自 replace;不删玩家昵称/公告/聊天里的 Emoji。
- **所有用户可见展示信息一律中文(强制)**:插件自己输出给玩家的文字必须是中文,包括游戏的英文默认名/占位/枚举/dev 名/内部标识都要转成中文再显示。
  - 例:公会默认名 `Unnamed Guild` → 展示为中文(如「队长」的公会 / 未命名公会);属性/工作/状态/类型等英文键要映射中文;新增功能的按钮/表头/提示/错误文案一律中文。
  - **例外(不翻译)**:玩家自己输入的内容(昵称、自定义公会名、公告、聊天),原样保留;稳定 ID / item_id / pal_dev_name / mission_id 作数据主键时不翻译(仅内部用,不直接展示;必须展示时旁边给中文名)。
  - 新增任何展示前自检:这张卡/这条消息里有没有玩家会看到的英文?有就补中文映射或回退中文。
- **1.0 在线数据优先 `/v1/api/game-data`**(GuildID/GuildName/坐标/Stage 按稳定 ID 合并),旧接口/存档为回退;存档解析失败须有可诊断原因。REST 仅限 LAN/容器内部。
- 游戏更新后优先重新生成+校验;数据生成失败不得覆盖上一版可用数据。
- 无真实存档验证时**不得声称完全兼容**。
- 提取工具逐步收进 `tools/game_data/`,不长期依赖仓库外隐藏目录。

## 开发方式

- 修改前先读相关代码和测试;先定位根因再改;小步、可运行、可回滚。
- 不删功能换测试通过;不改与当前任务无关的代码;不伪造测试结果。
- 缺少游戏目录/真实存档/UI 截图/素材时**必须明确说明**。
- 每阶段完成后运行:`python -m compileall -q .` / `ruff check .` / `pytest -q`。

## 文档 / 版本同步

以下改变时必须同步 README、CHANGELOG、测试:指令 / 配置 / 主题 / 游戏数据版本 / 存档兼容范围 / 素材生成方式 / 项目版本。

**版本一致性(测试强制)**:`metadata.yaml`、`pyproject.toml`、`main.py @register`、README 徽章、CHANGELOG 最新条目 必须完全相同。
