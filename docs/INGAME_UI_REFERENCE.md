## 关键结论:`docs/screenshots/` 不是游戏截图

已逐张核实:`docs/screenshots/*.png`(31 张)是**本插件自己渲染的 fantasy 卡片**(README 展示用),
**不是《幻兽帕鲁》游戏内 UI 截图**。例如:

- `status.png`:插件卡,含 ⚡FPS / 📅游戏天数 / ⏱运行时长 / 👥在线玩家 / ◆服务器负载 / CPU / 内存 —— 全是 Emoji + 主机指标,正是要被替换的**当前状态**。
- `paldex.png`:插件卡,含 🥚普通蛋 / 📈刷新 / 🎯捕获率 / 📏体型 / 📊基础数值 / 🐑牧场产出 / ⚔主动技能 / 🔨工作适性 / 🎁掉落物品 —— 属性徽章是**文字**(「无」「雷」)。

➡ 它们可作「改造前后对照」的**基线**,但**不能**当游戏原生 UI 的布局/配色参考。

## 缺失:游戏原生 UI 截图(一次性清单,请统一提供)

按 ingame 还原优先级排序。**核心 6 项**(阶段 D)务必先给:

### 核心(阶段 D 必需)
1. **帕鲁详情页**(属性面板:HP/攻击/防御/耐力/SAN、工作适性图标行、稀有度边框、性别、闪光标、主动技能列表元素图标)
2. **帕鲁箱 / 队伍**(格子布局、选中态、闪光/头目/浓缩标、等级条)
3. **背包 / 物品栏**(格子网格、稀有度底色、数量角标、分类标签)
4. **世界/服务器设置**(选项行排版、开关样式、数值滑条)
5. **属性相克表**(9 系图标、克制箭头/颜色)
6. **状态/异常图标**(灼烧/中毒/冰冻/骨折/饥饿/虚弱/精神 —— 帕鲁或角色状态栏)

### 次要(阶段 E)
7. 科技树 / 研究所(科技点图标、解锁态、锁定态)
8. 建造菜单(设施图标、材料消耗行)
9. 世界地图(标记/BOSS/快速旅行点图标、图例)
10. 任务日志(主线/支线标、目标勾选态)
11. 商店 / 商人(货币图标、价格行)
12. 公会面板(会长标、成员行)
13. 觉醒页(9 系晶石图标、觉醒机制 UI)
14. 突变 / 配种页(蛋图标、配种箭头、突变标记)
15. 竞技场 / 塔主 / 突袭 BOSS(段位/奖励图标)
16. 图鉴总览网格(编号、稀有度边框、未捕获态)

### 通用素材参考(任意页面即可体现)
- 面板/窗口**九宫格边框纹理**(border-image 用)
- 稀有度**边框配色**(普通→传说 各级颜色)
- 货币图标(金币/狗币/科技点/古代科技点)
- 元素 9 系图标、工作适性 14 图标(若能直接给「合成图鉴/图标表」一张更省事)

## 提供方式建议

- 手机竖屏截图优先(与 QQ 群阅读场景一致);PC 截图也可(仅取视觉语言,不照搬 1920×1080)。
- 能一次给「图标合集/图鉴页」的,优先于逐个页面(元素/工作/状态/货币图标集尤其如此)。
- 若能提供**游戏安装目录**(供 `tools/game_data/` 提取管线跑 texexport/bp_dump),则 element/work/status/currency 等图标可**直接从游戏文件提取**,无需截图,更准确 —— 这是最优路径。

> 在拿到核心截图或游戏目录前:阶段 B(Manifest + 解析器 + card_style 注册 + 测试骨架)**可先做**,不依赖截图;阶段 C 组件可用占位并明确标「缺参考」;阶段 D 起不得声称「高度还原」。
